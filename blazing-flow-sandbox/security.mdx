Blazing Flow Sandbox is designed for **untrusted user code**.

## Isolation Guarantees

- Sandboxed code runs in a restricted runtime (WASM/Pyodide).
- No direct network or filesystem access from sandboxed steps.
- No access to secrets or credentials inside the sandbox.

## Trust Boundary

- **User code** runs in the sandbox.
- **Services** run in trusted workers and own all external access.
- Only explicitly exposed service methods are callable from sandboxed steps.

## Best Practices

- Keep sensitive logic inside services.
- Validate inputs before passing them to services.
- Log and audit service calls from sandboxed code.
