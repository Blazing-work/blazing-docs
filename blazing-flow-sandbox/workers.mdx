Blazing Flow separates trusted and sandboxed execution to keep user code isolated.

## Trusted Execution

- Runs services and orchestration
- Holds credentials and external connections

## Sandboxed Execution

- Runs user‑provided code in WASM
- No direct network or filesystem access

Blazing routes work automatically based on how you define the step.
