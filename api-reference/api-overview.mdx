The Blazing Core API provides RESTful endpoints for managing multi-tenant edge infrastructure across multiple cloud providers.

**Base URL**: `https://api.blazing.work`
**API Docs**: `https://api.blazing.work/docs` (Swagger UI)

## Architecture

Each cluster gets an isolated environment with:
- **Kubernetes Namespace** (`c-{cluster_id}`)
- **Consul Cluster** (1-5 nodes for service mesh)
- **Edge Envoy Gateway** (LoadBalancer for external access)
- **Worker Deployments** (scalable compute across GCP, DFC, Akash)

## Authentication

All API requests require authentication using an API key in the `X-API-Key` header:

```bash
curl -H "X-API-Key: your-api-key" https://api.blazing.work/v1/organizations
```

### Admin Operations

Some operations require admin-level access using the `X-Admin-Key` header:

```bash
curl -H "X-Admin-Key: your-admin-key" https://api.blazing.work/v1/admin/organizations
```

## API Structure

The API follows a hierarchical structure:

```text
/v1/
├── admin/
│   └── organizations         # Create organizations (admin only)
├── organizations/
│   ├── {org_id}/
│   │   ├── projects/         # Project management
│   │   └── api-keys/         # API key management
│   └── {org_id}/projects/{project_id}/
│       └── clusters/         # Cluster management
└── health                    # Health check endpoint
```

## Response Format

All API responses follow a consistent JSON format:

### Success Response

```json
{
  "organization_id": "acme-corp",
  "name": "Acme Corporation",
  "created_at": "2025-11-17T10:00:00.000Z"
}
```

### Error Response

```json
{
  "detail": "Organization not found"
}
```

## Rate Limits

API requests are rate-limited based on your subscription plan:

- **Free**: 100 requests/minute
- **Pro**: 1,000 requests/minute
- **Enterprise**: Custom limits

## Pagination

List endpoints support pagination using query parameters:

```bash
curl "https://api.blazing.work/v1/organizations/acme-corp/projects?page=1&limit=50" \
  -H "X-API-Key: your-api-key"
```

## Async Operations

Some operations (like cluster creation) are asynchronous and return a task ID:

```json
{
  "cluster_id": "us-west-cluster",
  "status": "pending",
  "task_id": "task-12345"
}
```

Poll the cluster status endpoint to check progress:

```bash
curl "https://api.blazing.work/v1/organizations/acme-corp/projects/production/clusters/us-west-cluster" \
  -H "X-API-Key: your-api-key"
```
