Design your steps to be safe to retry and fail fast when inputs are invalid.

## Basic Pattern

```python
import asyncio
from blazing import Blazing

async def main():
    app = Blazing()

    @app.step
    async def risky(x: int, services=None):
        if x < 0:
            raise ValueError("x must be non-negative")
        return x * 2

    @app.workflow
    async def safe_workflow(x: int, services=None):
        try:
            return {"ok": True, "result": await risky(x, services=services)}
        except ValueError as e:
            return {"ok": False, "error": str(e)}

    await app.publish()
    print(await app.safe_workflow(-1).wait_result())

if __name__ == "__main__":
    asyncio.run(main())
```

## Best Practices

- Validate inputs early.
- Keep steps idempotent so retries are safe.
- Surface clear errors to callers.
