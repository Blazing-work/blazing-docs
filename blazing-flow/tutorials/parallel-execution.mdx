Blazing Batch automatically parallelizes task execution across distributed workers. Learn how to leverage this for maximum performance.

## Basic Parallel Execution

### Execute Multiple Tasks

```python
from blazing import Blazing

app = Blazing()

# Create multiple tasks
symbols = ["AAPL", "TSLA", "GOOGL", "MSFT", "AMZN"]
tasks = []

for symbol in symbols:
    unit = await app.create_route_task(
        "fetch_timeseries_route",
        symbol=symbol,
        points=50,
        delay=0
    )
    tasks.append(unit)

# Execute all tasks in parallel
results = await app.gather(tasks)

# Process results
for i, result in enumerate(results):
    print(f"{result['symbol']}: {result['count']} points")
```

## Parallelization Benefits

### Sequential vs Parallel

**Sequential execution** (4 tasks × 1s each = ~4 seconds):
```python
# DON'T DO THIS - runs one at a time
for symbol in symbols:
    unit = await app.create_route_task(...)
    result = await app.gather([unit])  # Waits before next iteration
```

**Parallel execution** (4 tasks × 1s each = ~1 second):
```python
# DO THIS - creates all tasks first, then waits
tasks = []
for symbol in symbols:
    tasks.append(await app.create_route_task(...))

# All tasks execute concurrently
results = await app.gather(tasks)
```

### Real-World Example

```python
import asyncio
import time
from blazing import Blazing

async def parallel_data_pipeline():
    app = Blazing()

    start_time = time.time()

    # Create 100 parallel tasks
    tasks = []
    for i in range(100):
        unit = await app.create_route_task(
            "fetch_timeseries_route",
            symbol=f"STOCK{i}",
            points=1000,
            delay=0
        )
        tasks.append(unit)

    # Execute all in parallel
    results = await app.gather(tasks)

    elapsed = time.time() - start_time
    print(f"✓ Processed {len(results)} tasks in {elapsed:.2f}s")

    await app.close()

asyncio.run(parallel_data_pipeline())
```

## Task with Delays

Even tasks with delays benefit from parallelization:

```python
import time

app = Blazing()
start_time = time.time()

# Create 4 tasks with 1-second delay each
tasks = []
for i in range(4):
    unit = await app.create_route_task(
        "fetch_timeseries_route",
        symbol=f"SYM{i}",
        points=10,
        delay=1  # 1 second delay per task
    )
    tasks.append(unit)

# Parallel: completes in ~1-2 seconds (not 4!)
await app.wait(tasks)

elapsed = time.time() - start_time
print(f"Parallel execution: {elapsed:.2f}s")
# Output: ~1.5s instead of 4s
```

## Best Practices

### 1. Batch Task Creation

```python
# Good: Create all tasks first
tasks = [
    await app.create_route_task("route", param=i)
    for i in range(100)
]
results = await app.gather(tasks)
```

### 2. Use gather() for Results

```python
# Efficient: Single gather call
results = await app.gather(tasks)

# Inefficient: Multiple individual gathers
results = [await app.gather([task]) for task in tasks]
```

### 3. Monitor Concurrency

```python
# Create tasks in batches for very large workloads
batch_size = 50

for i in range(0, len(all_tasks), batch_size):
    batch = all_tasks[i:i + batch_size]
    results = await app.gather(batch)
    process_results(results)
```
