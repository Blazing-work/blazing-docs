Learn advanced concurrency patterns for distributed task processing with Blazing Batch's CRDT-based queue system.

## CRDT Queue Architecture

Blazing Batch uses **CRDT (Conflict-free Replicated Data Types)** for safe distributed queue operations:

- **No duplicate processing** - Each task processed exactly once
- **Lock-free operations** - No distributed locks needed
- **Multi-node coordination** - Workers on different nodes coordinate automatically
- **Partition tolerance** - Works across network partitions

## Node-Based Queue Partitioning

### How It Works

Each API node writes to its own queue segment:

```python
# Node 1 writes to: queue:station-123:node-1
# Node 2 writes to: queue:station-123:node-2
# Node 3 writes to: queue:station-123:node-3
```

Workers scan all queue segments and dequeue operations safely.

### Example: Multi-Node Enqueue

```python
import os
from blazing_service.data_access.data_access import StationDAO

# Simulate different nodes
nodes = ["api-node-1", "api-node-2", "api-node-3"]

for node_id in nodes:
    os.environ['NODE_ID'] = node_id

    # Each node enqueues to its own segment
    await StationDAO.enqueue_non_blocking_operation(
        station_id="station-123",
        operation_id=f"op-from-{node_id}"
    )

# Result: 3 queue segments created
# - queue:station-123:api-node-1
# - queue:station-123:api-node-2
# - queue:station-123:api-node-3
```

## Race Condition Handling

### Concurrent Status Updates

Multiple workers can safely update operation status concurrently:

```python
import asyncio
from blazing_service.data_access.data_access import OperationDAO

async def concurrent_status_updates():
    """Multiple workers updating same operation."""

    operation_id = "op-12345"

    # Simulate 5 workers trying to claim the operation
    async def worker_claim(worker_id: int):
        success = await OperationDAO.try_claim_operation(
            operation_id,
            worker_id=f"worker-{worker_id}"
        )
        return (worker_id, success)

    # Race: Only one worker should successfully claim
    results = await asyncio.gather(*[
        worker_claim(i) for i in range(5)
    ])

    successful = [r for r in results if r[1]]

    assert len(successful) == 1, "Exactly one worker should claim operation"
    print(f"✓ Worker {successful[0][0]} claimed operation")
```

### Lock-Free Dequeuing

Workers dequeue operations without locks using atomic Redis operations:

```python
from blazing_service.data_access.data_access import StationDAO

async def lock_free_dequeue(station_id: str):
    """Dequeue next operation without blocking."""

    # Atomic operation - safe for concurrent workers
    operation_id = await StationDAO.dequeue_non_blocking_operation(
        station_id
    )

    if operation_id:
        print(f"✓ Dequeued: {operation_id}")
        return operation_id
    else:
        print("Queue empty")
        return None
```

## Multi-App Isolation

### Concurrent App Instances

Multiple Blazing app instances can run concurrently with full isolation:

```python
from blazing import Blazing

# Create multiple isolated apps
apps = [Blazing(app_id=f"app-{i}") for i in range(5)]

# Each app has isolated:
# - Queue namespace
# - Operation tracking
# - Worker pool

# Run tasks from all apps concurrently
all_tasks = []
for i, app in enumerate(apps):
    unit = await app.create_route_task(
        "fetch_timeseries_route",
        symbol=f"APP{i}-STOCK",
        points=100
    )
    all_tasks.append(unit)

# All apps process in parallel, fully isolated
results = await asyncio.gather(*[
    app.gather([task])
    for app, task in zip(apps, all_tasks)
])
```

### Cross-App Communication

Apps are isolated by default, but can share data through Redis:

```python
from blazing_service.data_access.data_access import CrossAppDAO

# App 1 publishes result
await CrossAppDAO.publish_result(
    channel="shared-channel",
    data={"result": "processing complete"}
)

# App 2 subscribes
async for message in CrossAppDAO.subscribe("shared-channel"):
    print(f"Received: {message}")
```

## Network Resilience

### Handling Network Partitions

Blazing Batch is designed to handle network issues gracefully:

```python
import asyncio

async def resilient_task_execution():
    """Execute tasks with network fault tolerance."""

    app = Blazing()

    try:
        unit = await app.create_route_task(
            "fetch_timeseries_route",
            symbol="AAPL",
            points=1000
        )

        # Set timeout for network issues
        result = await asyncio.wait_for(
            unit.result(),
            timeout=30.0
        )

        return result

    except asyncio.TimeoutError:
        # Network partition or slow worker
        print("✗ Task timeout - possible network issue")

        # Check if task eventually completed
        await asyncio.sleep(5)
        status = await app.check_task_status(unit)

        if status == "completed":
            print("✓ Task completed despite timeout")
            return await unit.result()

    finally:
        await app.close()
```

### Automatic Retry on Network Failure

```python
async def network_resilient_gather(app, units, max_retries=3):
    """Gather with automatic retry on network failures."""

    for attempt in range(max_retries):
        try:
            return await asyncio.wait_for(
                app.gather(units),
                timeout=60.0
            )
        except asyncio.TimeoutError:
            if attempt == max_retries - 1:
                raise

            wait_time = 2 ** attempt
            print(f"Network timeout, retry {attempt + 1}/{max_retries}")
            await asyncio.sleep(wait_time)
```

## Worker Lifecycle Management

### Coordinated Worker Startup

```python
from blazing_service.worker import Worker

async def coordinated_worker_pool(num_workers: int):
    """Start multiple workers in coordinated fashion."""

    workers = []

    for i in range(num_workers):
        worker = Worker(
            worker_id=f"worker-{i}",
            app_id="default"
        )
        workers.append(worker)

    # Start all workers concurrently
    await asyncio.gather(*[
        worker.start() for worker in workers
    ])

    print(f"✓ Started {len(workers)} coordinated workers")

    return workers
```

### Graceful Shutdown

```python
async def graceful_worker_shutdown(workers: list):
    """Shut down workers gracefully, completing in-flight tasks."""

    print("Initiating graceful shutdown...")

    # Signal workers to stop accepting new tasks
    for worker in workers:
        await worker.stop_accepting()

    # Wait for in-flight tasks to complete (max 30s)
    try:
        await asyncio.wait_for(
            asyncio.gather(*[
                worker.wait_for_completion()
                for worker in workers
            ]),
            timeout=30.0
        )
        print("✓ All tasks completed")
    except asyncio.TimeoutError:
        print("⚠ Timeout waiting for tasks, forcing shutdown")

    # Final cleanup
    for worker in workers:
        await worker.close()
```

## Advanced Patterns

### Fan-Out / Fan-In

Process tasks in parallel, then aggregate results:

```python
async def fan_out_fan_in(data_chunks: list):
    """Fan-out: process chunks in parallel, fan-in: aggregate."""

    app = Blazing()

    # Fan-out: Create tasks for all chunks
    units = []
    for i, chunk in enumerate(data_chunks):
        unit = await app.create_route_task(
            "process_chunk",
            chunk_id=i,
            data=chunk
        )
        units.append(unit)

    # Execute in parallel
    chunk_results = await app.gather(units)

    # Fan-in: Aggregate all results
    aggregated = await app.create_route_task(
        "aggregate_results",
        results=chunk_results
    )

    final_result = await aggregated.result()

    await app.close()
    return final_result
```

### Pipeline Processing

Chain multiple processing stages:

```python
async def multi_stage_pipeline(input_data: list):
    """Process data through multiple stages."""

    app = Blazing()

    # Stage 1: Data extraction
    extract_units = [
        await app.create_route_task("extract", item=item)
        for item in input_data
    ]
    extracted = await app.gather(extract_units)

    # Stage 2: Data transformation
    transform_units = [
        await app.create_route_task("transform", data=data)
        for data in extracted
    ]
    transformed = await app.gather(transform_units)

    # Stage 3: Data loading
    load_units = [
        await app.create_route_task("load", data=data)
        for data in transformed
    ]
    loaded = await app.gather(load_units)

    await app.close()
    return loaded
```

## Best Practices

1. **Use NODE_ID** - Set unique `NODE_ID` for each API instance
2. **Implement timeouts** - Always use timeouts for network operations
3. **Handle retries** - Implement exponential backoff for transient failures
4. **Monitor queue depth** - Track queue segments per station
5. **Graceful shutdown** - Wait for in-flight tasks before termination
6. **Isolate apps** - Use separate `app_id` for different workloads
