Learn how to create and execute distributed tasks using Blazing Batch's async API.

## Prerequisites

- Blazing Batch Python SDK installed
- Redis running (default: `localhost:6380`)
- Access to Blazing Batch API endpoint

## Quick Start

### 1. Initialize the App

```python
from blazing import Blazing

# Create a Blazing app instance
app = Blazing()
```

### 2. Create a Simple Task

```python
# Create a task to fetch time series data
unit = await app.create_route_task(
    "fetch_timeseries_route",
    symbol="AAPL",
    points=100,
    delay=0
)
```

### 3. Wait for Results

```python
# Gather results from the task
results = await app.gather([unit])
result = results[0]

print(f"Symbol: {result['symbol']}")
print(f"Data points: {result['count']}")
```

## Complete Example

Here's a complete working example:

```python
import asyncio
from blazing import Blazing

async def main():
    # Initialize app
    app = Blazing()

    # Create task
    unit = await app.create_route_task(
        "fetch_timeseries_route",
        symbol="AAPL",
        points=100,
        delay=0
    )

    # Get results
    results = await app.gather([unit])
    result = results[0]

    print(f"✓ Fetched {result['count']} points for {result['symbol']}")

    # Clean up
    await app.close()

if __name__ == "__main__":
    asyncio.run(main())
```

## Key Concepts

### Routes

Routes are pre-configured task endpoints that define:
- Task parameters
- Execution logic
- Return types

Common routes:
- `fetch_timeseries_route` - Fetch time series data
- `quick_lookup_route` - Quick data lookups
- Custom routes defined by your organization

### Units

A **unit** represents a single task execution:
- Created by `create_route_task()`
- Can be gathered for results
- Executed asynchronously by workers

### Gather vs Wait

```python
# gather() - Wait AND retrieve results
results = await app.gather([unit1, unit2])

# wait() - Just wait for completion (no results)
await app.wait([unit1, unit2])
```
