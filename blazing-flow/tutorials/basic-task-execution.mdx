Learn how to create and execute workflows using Blazing Flow's decorator-based API.

## Prerequisites

- Blazing Flow Python SDK installed (`pip install blazing`)
- Python 3.8 or higher
- Basic understanding of async/await in Python

## Quick Start

### 1. Initialize the App

```python
from blazing import Blazing
import asyncio

async def main():
    app = Blazing()  # Uses Blazing SaaS by default

if __name__ == "__main__":
    asyncio.run(main())
```

### 2. Define a Step

Steps are the building blocks of workflows. Each step is a unit of computation.

```python
from blazing import Blazing
import asyncio

async def main():
    app = Blazing()

    @app.step
    async def double(x: int, services=None):
        """Double a number."""
        return x * 2

if __name__ == "__main__":
    asyncio.run(main())
```

**Key points:**
- Use `@app.step` decorator
- Function must be `async def`
- Must accept `services=None` parameter for dependency injection

### 3. Create a Workflow

Workflows orchestrate multiple steps together.

```python
from blazing import Blazing
import asyncio

async def main():
    app = Blazing()

    @app.step
    async def double(x: int, services=None):
        """Double a number."""
        return x * 2

    @app.step
    async def add_ten(x: int, services=None):
        """Add 10 to a number."""
        return x + 10

    @app.workflow
    async def process_number(x: int, services=None):
        """Workflow: double then add 10."""
        doubled = await double(x, services=services)
        result = await add_ten(doubled, services=services)
        return result

if __name__ == "__main__":
    asyncio.run(main())
```

**Key points:**
- Use `@app.workflow` decorator for orchestration
- Call steps with `await` and forward `services` parameter
- Steps execute in the order you call them

### 4. Publish and Execute

```python
from blazing import Blazing
import asyncio

async def main():
    app = Blazing()

    @app.step
    async def double(x: int, services=None):
        return x * 2

    @app.step
    async def add_ten(x: int, services=None):
        return x + 10

    @app.workflow
    async def process_number(x: int, services=None):
        doubled = await double(x, services=services)
        result = await add_ten(doubled, services=services)
        return result

    # Publish workflow to Blazing
    await app.publish()

    # Execute and wait for result
    result = await app.process_number(5).wait_result()
    print(f"Result: {result}")  # Result: 20

if __name__ == "__main__":
    asyncio.run(main())
```

## Complete Example

Here's a complete working example:

```python
import asyncio
from blazing import Blazing

async def main():
    # Initialize app
    app = Blazing()

    # Define steps
    @app.step
    async def fetch_data(symbol: str, services=None):
        """Simulate fetching stock data."""
        return {"symbol": symbol, "price": 150.25}

    @app.step
    async def calculate_value(data: dict, shares: int, services=None):
        """Calculate total value."""
        return data["price"] * shares

    # Define workflow
    @app.workflow
    async def portfolio_value(symbol: str, shares: int, services=None):
        """Calculate portfolio value for a stock."""
        data = await fetch_data(symbol, services=services)
        value = await calculate_value(data, shares, services=services)
        return {
            "symbol": symbol,
            "shares": shares,
            "value": value
        }

    # Publish workflow
    await app.publish()

    # Execute workflow
    result = await app.portfolio_value("AAPL", 100).wait_result()
    print(f"Portfolio: {result['symbol']}")
    print(f"Shares: {result['shares']}")
    print(f"Total Value: ${result['value']:.2f}")

if __name__ == "__main__":
    asyncio.run(main())
```

**Output:**
```
Portfolio: AAPL
Shares: 100
Total Value: $15025.00
```

## Key Concepts

### Steps vs Workflows

| Aspect | Step | Workflow |
|--------|------|----------|
| Purpose | Single unit of work | Orchestrate multiple steps |
| Decorator | `@app.step` | `@app.workflow` |
| Calls other steps | No | Yes |
| Callable from | Workflows only | External code or other workflows |

### Services Parameter

All steps and workflows must accept `services=None`:

```python
@app.step
async def my_step(x: int, services=None):
    # Access injected services if needed
    if services and services.connectors:
        db = services.connectors.postgres
        # Use database...
    return x * 2
```

The `services` parameter enables:
- Database connector injection
- External service access
- Testing with mocked services

### Execution Methods

```python
# wait_result() - Wait for completion and get result
result = await app.my_workflow(5).wait_result()

# wait() - Wait for completion without getting result
await app.my_workflow(5).wait()

# Direct call (for testing/development)
result = await my_workflow(5, services=None)
```
