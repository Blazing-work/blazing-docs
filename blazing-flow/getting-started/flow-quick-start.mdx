Get started with Blazing Flow and build your first asynchronous pipeline in just a few minutes.

## Prerequisites

- Python 3.11 or higher
- Redis or KeyDB server
- Basic understanding of Python async/await

## Installation

Install Blazing Flow using pip or uv:

```bash
# Using pip
pip install blazing

# Using uv (recommended)
uv pip install blazing
```

Or install from source:

```bash
git clone https://github.com/your-org/blazing.git
cd blazing
uv sync
source .venv/bin/activate
```

## Start Redis

Blazing Flow requires a Redis server for state management. You can use Docker to quickly spin one up:

```bash
# Pull the Redis Stack image
docker pull redis/redis-stack

# Run Redis on port 6380
docker run --name blazing_redis -p 6380:6379 -d redis/redis-stack
```

## Your First Pipeline

Let's create a simple data processing pipeline that demonstrates the core concepts of Blazing Flow.

### Step 1: Define a Service

Services encapsulate your business logic and can be reused across different stations:

```python
from blazing import BaseService

class DataProcessor(BaseService):
    """A service for processing data"""

    async def fetch_data(self, source_id: str):
        """Fetch data from a source"""
        # Simulate fetching data
        return {"id": source_id, "data": [1, 2, 3, 4, 5]}

    async def transform_data(self, data: dict):
        """Transform the data"""
        # Apply some transformation
        transformed = {
            "id": data["id"],
            "sum": sum(data["data"]),
            "count": len(data["data"])
        }
        return transformed

    async def save_result(self, result: dict):
        """Save the processed result"""
        # Save to database, file, or API
        print(f"Saved result: {result}")
        return result
```

### Step 2: Create a Blazing Application

Initialize your Blazing application with Redis configuration:

```python
from blazing import Blazing

# Create the app
app = Blazing(
    redis_config={
        'host': 'localhost',
        'port': 6380
    },
    services=[DataProcessor]
)
```

### Step 3: Define Stations

Steps are individual processing units in your pipeline:

```python
@app.step
async def fetch_station(source_id: str, services):
    """Step that fetches data"""
    processor = services['DataProcessor']
    data = await processor.fetch_data(source_id)
    return data

@app.step
async def transform_station(data: dict, services):
    """Step that transforms data"""
    processor = services['DataProcessor']
    result = await processor.transform_data(data)
    return result

@app.step
async def save_station(result: dict, services):
    """Step that saves the result"""
    processor = services['DataProcessor']
    final = await processor.save_result(result)
    return final
```

### Step 4: Define a Route

Routes orchestrate stations into a complete workflow:

```python
@app.workflow
async def data_pipeline(source_id: str):
    """Complete data processing pipeline"""
    # Fetch data
    data = await fetch_station(source_id)

    # Transform data
    result = await transform_station(data)

    # Save result
    final = await save_station(result)

    return final
```

### Step 5: Start the System

Put it all together and start your application:

```python
import asyncio

async def main():
    # Initialize and start the system
    await app.start()

    # Execute the pipeline
    result = await data_pipeline("source_123")
    print(f"Pipeline result: {result}")

    # Stop the system
    await app.stop()

if __name__ == "__main__":
    asyncio.run(main())
```

## Complete Example

Here's the complete code in one file:

```python
import asyncio
from blazing import Blazing, BaseService

class DataProcessor(BaseService):
    async def fetch_data(self, source_id: str):
        return {"id": source_id, "data": [1, 2, 3, 4, 5]}

    async def transform_data(self, data: dict):
        return {
            "id": data["id"],
            "sum": sum(data["data"]),
            "count": len(data["data"])
        }

    async def save_result(self, result: dict):
        print(f"Saved result: {result}")
        return result

app = Blazing(
    redis_config={'host': 'localhost', 'port': 6380},
    services=[DataProcessor]
)

@app.step
async def fetch_station(source_id: str, services):
    processor = services['DataProcessor']
    return await processor.fetch_data(source_id)

@app.step
async def transform_station(data: dict, services):
    processor = services['DataProcessor']
    return await processor.transform_data(data)

@app.step
async def save_station(result: dict, services):
    processor = services['DataProcessor']
    return await processor.save_result(result)

@app.workflow
async def data_pipeline(source_id: str):
    data = await fetch_station(source_id)
    result = await transform_station(data)
    final = await save_station(result)
    return final

async def main():
    await app.start()
    result = await data_pipeline("source_123")
    print(f"Pipeline result: {result}")
    await app.stop()

if __name__ == "__main__":
    asyncio.run(main())
```

## Running the Example

Save the code to `my_pipeline.py` and run it:

```bash
python my_pipeline.py
```

You should see output like:

```
Saved result: {'id': 'source_123', 'sum': 15, 'count': 5}
Pipeline result: {'id': 'source_123', 'sum': 15, 'count': 5}
```
