Blazing Flow is a managed workflow engine for Python. You define **steps**, **workflows**, and **services** with simple decorators, publish them, and execute runs from your application or API layer.

## Get Started

- **[Quick Start](/docs/flow-quick-start)** – Build your first pipeline in minutes
- **[Guide](/docs/flow-guide)** – Core concepts and practical patterns

## Why Blazing Flow

- **Real Python control flow** – loops, conditionals, dynamic branching
- **Reusable services** – shared logic and pooled external connections
- **Trusted + sandboxed execution** – safe multi‑tenant workloads
- **Batching and scheduling** – built into the workflow model
- **Custom images** – system dependencies when you need them

## When to Use Blazing Flow

- Multi‑step pipelines with branching logic
- Long‑running or fan‑out workloads
- Stateful services that manage external resources
- Multi‑tenant systems where isolation matters

## Quick Example

```python
import asyncio
from blazing import Blazing

app = Blazing()

@app.step
async def fetch_data(source_id: str, services=None):
    return {"id": source_id, "data": [1, 2, 3]}

@app.step
async def transform_data(raw_data: dict, services=None):
    return {"id": raw_data["id"], "result": sum(raw_data["data"])}

@app.workflow
async def data_pipeline(source_id: str, services=None):
    data = await fetch_data(source_id, services=services)
    return await transform_data(data, services=services)

async def main():
    await app.publish()
    result = await app.data_pipeline(source_id="123").wait_result()
    print(result)

if __name__ == "__main__":
    asyncio.run(main())
```
