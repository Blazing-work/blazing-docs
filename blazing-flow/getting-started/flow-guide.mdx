Learn how to build resilient, scalable async pipelines with Blazing Flow.

## What is Blazing Flow?

Blazing Flow is a managed, developer‑first workflow engine. You define steps and workflows in Python, publish them, and execute runs from your application or API layer.

### Key Features

- **Declarative pipelines** with simple Python decorators
- **Resilient execution** with retry‑friendly patterns
- **Trusted + sandboxed execution** for safe multi‑tenant workloads
- **Batching and scheduling** built into the workflow model
- **Custom executor images** when you need system dependencies

## Quick Example

```python
from blazing import Blazing

app = Blazing()

@app.step
async def fetch_data(source_id: str, services=None):
    return {"id": source_id, "data": [1, 2, 3]}

@app.step
async def transform_data(data: dict, services=None):
    return {"id": data["id"], "result": sum(data["data"])}

@app.workflow
async def data_pipeline(source_id: str, services=None):
    data = await fetch_data(source_id, services=services)
    return await transform_data(data, services=services)

async def main():
    await app.publish()
    result = await app.data_pipeline(source_id="123").wait_result()
    print(result)
```

## Core Concepts

### Steps

Steps are single‑responsibility functions. Use them for reusable units of work.

### Workflows

Workflows orchestrate steps. They are the entry points you call from your app or API layer.

### Services

Services encapsulate business logic and external I/O. They are injected into steps via the `services` parameter.

## Dynamic Batching

Use `@batched` to combine individual calls into a single batch execution.

```python
from blazing import batched, BatchConfig

@batched(BatchConfig(max_batch_size=32, wait_ms=50))
@app.step
async def embed(texts: list[str], services=None):
    return await services["EmbeddingService"].encode(texts)
```

## Scheduled Workflows

```python
from blazing import Cron, Period

@app.workflow(schedule=Cron("0 2 * * *"))
async def nightly_report(services=None):
    ...

@app.workflow(schedule=Period(minutes=30))
async def sync_job(services=None):
    ...
```

## Custom Executor Images

```python
from blazing import Image

video_image = (
    Image.executor()
    .apt_install("ffmpeg")
    .pip_install("opencv-python")
    .env(MODEL_PATH="/models")
)

@app.step(image=video_image)
async def process_video(url: str, services=None):
    ...
```
