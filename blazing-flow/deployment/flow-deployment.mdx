Deploy Blazing Flow workflows to production with confidence using best practices for deployment, versioning, and rollback strategies.

## Deployment Methods

### CLI Deployment

Deploy workflows using the Blazing CLI:

```bash
# Deploy workflow
blazing flow deploy workflow.py \
  --name production-workflow \
  --environment production

# Deploy with configuration
blazing flow deploy workflow.py \
  --config flow.yaml \
  --environment production

# Deploy specific version
blazing flow deploy workflow.py \
  --version 2.1.0
```

### CI/CD Deployment

#### GitHub Actions

```yaml
# .github/workflows/deploy-flow.yml
name: Deploy Flow Workflow

on:
  push:
    branches: [main]
    paths: ['workflows/**']

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Install Blazing CLI
        run: npm install -g @blazing/cli

      - name: Authenticate
        run: blazing login --token ${{ secrets.BLAZING_TOKEN }}

      - name: Deploy to staging
        run: blazing flow deploy workflows/main.py --environment staging

      - name: Run tests
        run: blazing flow test workflows/main.py --environment staging

      - name: Deploy to production
        if: success()
        run: blazing flow deploy workflows/main.py --environment production
```

Deploy with Terraform:


## Versioning

### Semantic Versioning

Use semantic versioning for workflows:

```python
# workflow.py
from blazing import Blazing
import asyncio

async def main():
    app = Blazing()

    @app.step
    async def process_data(data: dict, services=None):
        """Process data."""
        return {"processed": True, "data": data}

    @app.workflow
    async def data_pipeline(data: dict, services=None):
        """Main data pipeline workflow.

        Version: 2.1.0
        2.1.0 -> 2.1.1: Bug fix
        2.1.0 -> 2.2.0: New feature (backward compatible)
        2.1.0 -> 3.0.0: Breaking change
        """
        result = await process_data(data, services=services)
        return result

    await app.publish()

if __name__ == "__main__":
    asyncio.run(main())
```

### Version Management

```bash
# List versions
blazing flow versions production-workflow

# Deploy specific version
blazing flow deploy workflow.py --version 2.1.0

# Rollback to previous version
blazing flow rollback production-workflow --to-version 2.0.0
```

## Deployment Strategies

### Blue-Green Deployment

Deploy new version alongside old version:

```yaml
# flow.yaml
deployment:
  strategy: blue-green

  # New version (green)
  green:
    version: "2.1.0"
    traffic: 0  # Start with no traffic

  # Old version (blue)
  blue:
    version: "2.0.0"
    traffic: 100

  # Gradual traffic shift
  traffic_shift:
    - step: 1
      green_traffic: 10
      duration: 10m

    - step: 2
      green_traffic: 50
      duration: 20m

    - step: 3
      green_traffic: 100
      duration: 10m
```

### Canary Deployment

Gradually roll out to subset of users:

```yaml
deployment:
  strategy: canary

  # Canary configuration
  canary:
    version: "2.1.0"

    # Traffic percentage
    stages:
      - percentage: 5
        duration: 30m
        success_criteria:
          error_rate: < 0.01
          latency_p99: < 500ms

      - percentage: 25
        duration: 1h
        success_criteria:
          error_rate: < 0.01

      - percentage: 100
        success_criteria:
          error_rate: < 0.01

    # Automatic rollback
    rollback:
      on_failure: true
      threshold:
        error_rate: 0.05  # Rollback if >5% errors
```

## Custom Executor Images

You can customize the executor image per step or workflow using `Image.executor()`.

```python
from blazing import Image

gpu_image = (
    Image.executor()
    .apt_install("ffmpeg")
    .pip_install("torch", "transformers")
)

@app.step(image=gpu_image)
async def embed(text: str, services=None):
    ...
```

**Note:** `set_executor_image()` and `ExecutorImage` are deprecated in favor of
`Image.executor()` and per‑step/per‑workflow `image=`.

### Graceful Shutdown

Handle shutdown gracefully:

```python
from blazing import Blazing
import asyncio

async def main():
    app = Blazing()

    @app.step
    async def cleanup(services=None):
        """Cleanup on shutdown."""
        # Finish in-flight tasks
        # await app.wait_for_tasks()

        # Close connections
        # await db.close()
        # await cache.close()

        # Save state
        # Checkpointing is only available for notebook sessions (not workflows)

        return {"status": "cleanup_complete"}

    @app.workflow
    async def shutdown_handler(services=None):
        """Handle graceful shutdown."""
        result = await cleanup(services=services)
        return result

    await app.publish()

if __name__ == "__main__":
    asyncio.run(main())
```

## Monitoring Deployment

### Deployment Metrics

Track deployment health:

```bash
# View deployment status
blazing flow deployment status production-workflow

# Monitor metrics during deployment
blazing flow metrics production-workflow \
  --live \
  --filter deployment

# View deployment logs
blazing flow logs production-workflow \
  --since 10m \
  --follow
```

### Rollback Procedures

Automated rollback on failures:

```yaml
deployment:
  rollback:
    # Automatic rollback
    automatic:
      enabled: true

      # Rollback triggers
      triggers:
        - error_rate > 0.05
        - latency_p99 > 1000ms
        - health_check_failures > 3

    # Manual rollback
    manual:
      retain_versions: 5  # Keep last 5 versions
```

## Multi-Environment Deployment

### Environment Configuration

```yaml
# environments/production.yaml
environment: production

resources:
  cpu: 4
  memory: 8GB

concurrency:
  max_concurrent: 100

state:
  storage: postgres
  connection: postgresql://prod-db/flow

monitoring:
  metrics:
    enabled: true
    exporters: [prometheus, datadog]
```

```yaml
# environments/staging.yaml
environment: staging

resources:
  cpu: 2
  memory: 4GB

concurrency:
  max_concurrent: 20

state:
  storage: postgres
  connection: postgresql://staging-db/flow
```

### Deploy to Multiple Environments

```bash
# Deploy to staging
blazing flow deploy workflow.py \
  --config environments/staging.yaml

# Promote to production
blazing flow promote workflow.py \
  --from staging \
  --to production
```
