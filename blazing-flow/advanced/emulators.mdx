Blazing ships local emulators so you can develop and test without cloud infrastructure.

## What You Get

Local emulators let you spin up a lightweight development stack with the same APIs you use in production.

Common emulators include:

- Registry (image storage)
- Build (image builds)
- Volume (persistent files)
- Secrets (credentials)
- Scheduler (cron/periodic)
- Dict (key‑value)
- Queue (jobs)
- LLM (mock responses)
- Bucket mount (object storage)

## Quick Start

```python
from blazing.local import LocalStack

stack = LocalStack()
await stack.start()

# Example: create a volume
volume = await stack.create_volume("my-data", token="my-token")
await volume.put_file("/config.json", b'{}')
await volume.commit()

await stack.stop()
```

## Best Practices

- Use emulators for unit and integration tests.
- Keep production credentials out of local configs.
- Treat emulator data as ephemeral.
