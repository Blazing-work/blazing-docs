## `@app.endpoint()`

Expose a workflow as an HTTP/WebSocket endpoint.

```python
def endpoint(
    path: str,
    method: str = "POST",
    auth: Optional[Callable] = None,
    enable_websocket: bool = False,
    response_mode: str = "async",
    timeout: Optional[int] = None,
    middleware: Optional[List] = None,
):
    ...
```

### Key Parameters

- `path`: URL path (e.g., `/calculate`)
- `method`: HTTP method (default `POST`)
- `auth`: Auth handler or strategy
- `enable_websocket`: Adds `{path}/ws`
- `response_mode`: `async`, `sync`, `stream`, or `webhook`
- `timeout`: Sync‑mode timeout in seconds
- `middleware`: Per‑endpoint middleware list

### Example

```python
@app.endpoint(path="/calculate")
@app.workflow
async def calculate(x: int, y: int, services=None):
    return x + y
```

## `create_asgi_app()`

Create a FastAPI app from your endpoints.

```python
async def create_asgi_app(
    blazing_app: Blazing,
    title: str = "Blazing Workflow API",
    description: str = "",
    version: str = "1.0.0",
    enable_cors: bool = True,
    framework: str = "fastapi",
    cors_origins: Optional[List[str]] = None,
    global_middleware: Optional[List] = None,
) -> FastAPI:
    ...
```

### Example

```python
fastapi_app = await create_asgi_app(app, title="My API")
```

## Notes

- `@app.endpoint` must wrap `@app.workflow`.
- Request fields are inferred from the workflow signature.
- The `services` parameter is excluded from request models.
