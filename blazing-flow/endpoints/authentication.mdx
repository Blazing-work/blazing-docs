Protect your endpoints with simple, explicit auth strategies.

## Quick Start: API Keys

```python
from blazing import Blazing
from blazing.auth import APIKeyAuth

app = Blazing()

@app.endpoint(path="/secure", auth=APIKeyAuth(keys=["<your-key>"]))
@app.workflow
async def secure_workflow(data: str, services=None):
    return {"result": data.upper()}
```

## JWT Authentication

```python
from blazing.auth import JWTAuth

auth = JWTAuth(secret="<jwt-secret>")

@app.endpoint(path="/secure", auth=auth)
@app.workflow
async def secure_workflow(data: str, services=None):
    return {"ok": True}
```

## Multiple Strategies

You can accept multiple auth strategies by passing a list:

```python
from blazing.auth import JWTAuth, APIKeyAuth

@app.endpoint(path="/secure", auth=[JWTAuth(secret="<jwt-secret>"), APIKeyAuth(keys=["<key>"])])
@app.workflow
async def secure_workflow(data: str, services=None):
    return {"ok": True}
```

## Custom Authentication

You can define your own auth handler and return a boolean:

```python
from fastapi.security import HTTPAuthorizationCredentials

async def verify_api_key(credentials: HTTPAuthorizationCredentials) -> bool:
    return credentials and credentials.credentials == "<expected-key>"

@app.endpoint(path="/secure", auth=verify_api_key)
@app.workflow
async def secure_workflow(data: str, services=None):
    return {"ok": True}
```

## Best Practices

- Keep secrets in your platform’s secret manager.
- Rotate API keys regularly.
- Prefer short‑lived JWTs for user‑level auth.
