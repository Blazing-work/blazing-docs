Enable real‑time updates for workflows with WebSocket endpoints.

## Enable WebSocket

```python
@app.endpoint(path="/calculate", enable_websocket=True)
@app.workflow
async def calculate(x: int, y: int, services=None):
    return x + y
```

This creates:

- `POST /calculate` for job creation
- `WS /calculate/ws` for streaming job updates

## Client Example (JavaScript)

```javascript
const ws = new WebSocket('ws://localhost:8080/calculate/ws');

ws.onopen = () => {
  ws.send(JSON.stringify({x: 10, y: 20}));
};

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);
  if (msg.type === "result") {
    console.log("Result:", msg.result);
    ws.close();
  }
};
```

## Message Types (High Level)

You will receive events such as:

- `job_created`
- `status_update`
- `result`
- `error`

## Authentication

Use the same auth strategies as HTTP endpoints. If you attach `auth=` to the endpoint, WebSocket connections are authenticated the same way.
