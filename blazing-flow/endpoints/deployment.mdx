Deploy your **API layer** (FastAPI, Django, Flask, etc.) the same way you deploy any web service. The only Blazing‑specific requirement is that your service can reach the Blazing API and authenticate with a token.

## Minimal Production Checklist

- Set `BLAZING_API_URL` and `BLAZING_API_TOKEN` as environment variables.
- Run behind your standard HTTP stack (load balancer, TLS, CDN as needed).
- Configure health checks for your API process.
- Use your platform’s secret manager for tokens.

## Example: FastAPI

```python
import os
from blazing import Blazing
from fastapi import FastAPI

app = FastAPI()
blazing = Blazing(
    api_url=os.environ["BLAZING_API_URL"],
    api_token=os.environ["BLAZING_API_TOKEN"],
)
```

## Deployment Options

Choose the deployment approach that matches your stack:

- Container platforms (ECS/Fargate, Kubernetes, Nomad)
- PaaS (Render, Fly.io, Heroku)
- Serverless web runtimes (where supported)

## Notes

Blazing Flow handles execution and scaling of workflows. Your API layer only needs to forward requests and handle responses.
