# Deployment

Deploy Blazing Flow Endpoints to production environments.

## Docker Compose

### docker-compose.yml

```yaml
version: '3.8'

services:
  api:
    build: .
    ports:
      - "8080:8080"
    environment:
      - BLAZING_API_URL=http://blazing-backend:8000
      - BLAZING_API_TOKEN=${API_TOKEN}
    command: uvicorn main:fastapi_app --host 0.0.0.0 --port 8080
    depends_on:
      - blazing-backend
    restart: unless-stopped

  blazing-backend:
    image: blazing/backend:latest
    ports:
      - "8000:8000"
    environment:
      - REDIS_URL=redis://redis:6379
    depends_on:
      - redis

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
```

### Dockerfile

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["uvicorn", "main:fastapi_app", "--host", "0.0.0.0", "--port", "8080"]
```

### Deploy

```bash
docker-compose up -d
```

## Kubernetes

### deployment.yaml

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: blazing-api
spec:
  replicas: 3
  selector:
    matchLabels:
      app: blazing-api
  template:
    metadata:
      labels:
        app: blazing-api
    spec:
      containers:
      - name: api
        image: your-registry/blazing-api:latest
        ports:
        - containerPort: 8080
        env:
        - name: BLAZING_API_URL
          value: "http://blazing-backend:8000"
        - name: BLAZING_API_TOKEN
          valueFrom:
            secretKeyRef:
              name: blazing-secrets
              key: api-token
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 30
        readinessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 10
---
apiVersion: v1
kind: Service
metadata:
  name: blazing-api
spec:
  selector:
    app: blazing-api
  ports:
  - port: 80
    targetPort: 8080
  type: LoadBalancer
```

### Deploy

```bash
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
```

## Production with Gunicorn

For production deployments, use Gunicorn with Uvicorn workers:

```bash
gunicorn main:fastapi_app \
  --workers 4 \
  --worker-class uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8080 \
  --access-logfile - \
  --error-logfile - \
  --log-level info
```

### Systemd Service

```ini
# /etc/systemd/system/blazing-api.service
[Unit]
Description=Blazing Flow Endpoints API
After=network.target

[Service]
Type=notify
User=www-data
Group=www-data
WorkingDirectory=/opt/blazing-api
Environment="BLAZING_API_URL=http://localhost:8000"
Environment="BLAZING_API_TOKEN=your-token"
ExecStart=/usr/local/bin/gunicorn main:fastapi_app \
    --workers 4 \
    --worker-class uvicorn.workers.UvicornWorker \
    --bind 0.0.0.0:8080
Restart=always

[Install]
WantedBy=multi-user.target
```

Start service:

```bash
sudo systemctl enable blazing-api
sudo systemctl start blazing-api
```

## AWS Lambda / Serverless

### Using Mangum

Install:

```bash
pip install mangum
```

Wrap FastAPI app:

```python
from mangum import Mangum

# Your FastAPI app
fastapi_app = await create_asgi_app(app)

# Wrap for Lambda
handler = Mangum(fastapi_app)
```

### serverless.yml

```yaml
service: blazing-api

provider:
  name: aws
  runtime: python3.11
  region: us-east-1
  environment:
    BLAZING_API_URL: ${env:BLAZING_API_URL}
    BLAZING_API_TOKEN: ${env:BLAZING_API_TOKEN}

functions:
  api:
    handler: main.handler
    events:
      - httpApi: '*'
```

Deploy:

```bash
serverless deploy
```

## Nginx Reverse Proxy

```nginx
upstream blazing_api {
    server localhost:8080;
}

server {
    listen 80;
    server_name api.example.com;

    # Redirect HTTP to HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.example.com;

    ssl_certificate /etc/letsencrypt/live/api.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.example.com/privkey.pem;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header X-Content-Type-Options "nosniff" always;

    # Rate limiting
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;
    limit_req zone=api_limit burst=20 nodelay;

    location / {
        proxy_pass http://blazing_api;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket support
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        # Timeouts
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # Health check
    location /health {
        proxy_pass http://blazing_api/health;
        access_log off;
    }
}
```

## Environment Variables

### Required

| Variable | Description | Example |
|----------|-------------|---------|
| `BLAZING_API_URL` | Blazing Flow backend URL | `http://localhost:8000` |
| `BLAZING_API_TOKEN` | API authentication token | `your-secret-token` |

### Optional

| Variable | Description | Default |
|----------|-------------|---------|
| `LOG_LEVEL` | Logging level | `INFO` |
| `WORKERS` | Number of worker processes | `4` |
| `MAX_REQUESTS` | Restart workers after N requests | `1000` |
| `MAX_REQUESTS_JITTER` | Jitter for worker restart | `100` |

### Example .env file

```bash
BLAZING_API_URL=http://localhost:8000
BLAZING_API_TOKEN=your-secret-token
LOG_LEVEL=INFO
WORKERS=4
MAX_REQUESTS=1000
```

## Scaling

### Horizontal Scaling

Add more replicas:

```bash
# Kubernetes
kubectl scale deployment blazing-api --replicas=5

# Docker Compose
docker-compose up -d --scale api=5
```

### Vertical Scaling

Increase resources:

```yaml
# Kubernetes
resources:
  requests:
    memory: "1Gi"
    cpu: "1000m"
  limits:
    memory: "2Gi"
    cpu: "2000m"
```

### Auto-Scaling (Kubernetes)

```yaml
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: blazing-api-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: blazing-api
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
  - type: Resource
    resource:
      name: memory
      target:
        type: Utilization
        averageUtilization: 80
```

## Performance Tuning

### Worker Configuration

```python
# Optimal workers = (2 x CPU cores) + 1
workers = (2 * os.cpu_count()) + 1

gunicorn main:fastapi_app \
  --workers $workers \
  --worker-class uvicorn.workers.UvicornWorker
```

### Connection Pooling

```python
# Configure connection pool for Blazing backend
app = Blazing(
    api_url="http://localhost:8000",
    api_token="token",
    max_connections=100,  # Connection pool size
    timeout=30            # Request timeout
)
```

### Caching

```python
from functools import lru_cache

@lru_cache(maxsize=1000)
async def expensive_operation(x: int) -> int:
    # Cached result
    return x * 2
```

## Troubleshooting

### High Memory Usage

```bash
# Check memory per worker
ps aux | grep gunicorn

# Reduce workers or restart after fewer requests
gunicorn main:fastapi_app \
  --workers 2 \
  --max-requests 500
```

### Connection Timeouts

```nginx
# Increase Nginx timeouts
proxy_connect_timeout 120s;
proxy_send_timeout 120s;
proxy_read_timeout 120s;
```

### WebSocket Issues

```nginx
# Ensure WebSocket headers are set
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
```
