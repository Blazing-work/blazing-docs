Blazing Core supports multiple cloud providers, giving you flexibility, cost optimization, and vendor independence.

## Supported Providers

### GCP (Google Cloud Platform)
Enterprise-grade reliability with cost-effective Standard tier networking.

**Best For:**
- Production control planes
- Low-latency applications
- Compliance-sensitive workloads

**Key Features:**
- GKE Standard (spot instances for 60-90% savings)
- 17x cheaper egress ($0.005/GB vs $0.085/GB Premium)
- Global network infrastructure
- Managed certificates and load balancing

### DFC (Digital Frontier Cloud)
Curated, cost-effective infrastructure on Akash Network.

**Best For:**
- Burst workers
- Cost-optimized workloads
- Stateless services with replication

**Key Features:**
- Zero egress fees
- ~60% cheaper than GCP
- Curated high-reliability providers
- Same Kubernetes interface

### Akash Network
Global decentralized compute marketplace.

**Best For:**
- Emergency overflow capacity
- Geographic diversity
- Cost-sensitive batch jobs

**Key Features:**
- Zero egress fees
- 70-80% cheaper than hyperscale clouds
- Providers in 50+ countries
- No long-term contracts

### Coming Soon

- **AWS EKS**: Amazon Elastic Kubernetes Service
- **Azure AKS**: Azure Kubernetes Service
- **Hybrid On-Prem**: Your own data centers

## Provider Status

Define your relationship with each provider using the `status` field:

```yaml
providers:
  gcp:
    status: prefer     # 1st choice
  dfc:
    status: allow      # 2nd choice
  akash:
    status: fallback   # Last resort
```

### Status Semantics

| Status | Priority | When Used | Use Case |
|--------|----------|-----------|----------|
| **prefer** | 1st | Always attempted first | Primary infrastructure |
| **allow** | 2nd | Used when preferred unavailable | Acceptable alternatives |
| **fallback** | 3rd | Emergency only | Burst overflow capacity |

## Selection Algorithm

The server scores providers based on multiple factors:

```python
def score_provider(provider, requirements):
    score = 0

    # Capacity availability (40 points)
    if provider.has_capacity(requirements):
        score += 40

    # Cost efficiency (weighted by strategy)
    cost_score = (100 - provider.relative_cost) * 0.6
    score += cost_score * (requirements.strategy.cost / 100)

    # Latency optimization (weighted by strategy)
    if provider.region in requirements.preferred_regions:
        score += requirements.strategy.latency

    # Compliance match (weighted by strategy)
    if provider.meets_compliance(requirements.controls):
        score += requirements.strategy.compliance

    # Feature availability (20 points or -30 penalty)
    if all_features_available(provider, requirements):
        score += 20
    else:
        score -= 30

    return score
```

### Cost Optimization Strategy

**Best Practice**: Use multiple providers strategically:

```yaml
providers:
  gcp:
    status: prefer          # Reliable baseline (3 replicas)
    node_pools:
      baseline: standard    # min_replicas on on-demand
      burst: spot           # Burst on spot (60-90% savings)

  dfc:
    status: prefer          # Cost-effective burst (7+ replicas)

  akash:
    status: fallback        # Emergency overflow
```

**Result**:
- Baseline (3 replicas): GCP on-demand ($40/month)
- Burst (7 replicas): GCP spot + DFC ($35/month)
- Total: **$75/month** vs $150/month all-GCP-standard (50% savings)

## Network Tiers

### GCP Network Tiers

```yaml
providers:
  gcp:
    network_tier: Standard | Premium
```

| Tier | Egress Cost | IPv6 | Latency | Use Case |
|------|-------------|------|---------|----------|
| **Standard** | $0.005/GB | ❌ No | Regional | Cost-optimized (default) |
| **Premium** | $0.085/GB | ✅ Yes | Global | Low-latency worldwide |

**Important**: Standard tier = **17x cheaper egress**. Use Premium only for truly global, latency-sensitive applications.

## Public Gateway

Only **one provider** should have the public gateway enabled:

```yaml
providers:
  gcp:
    status: prefer
    public_gateway:
      enabled: true         # External traffic enters here
      static_ip: auto

      http:
        enabled: true
        port: 443

      gateway:
        replicas: 1
        resources:
          cpu: 500m
          memory: 1Gi

  dfc:
    status: prefer
    public_gateway:
      enabled: false        # Workers only, no gateway
```

**Pricing**: $49/month (includes static IP, certs, HTTP/HTTPS ingress)

**Recommendation**: Enable on GCP for maximum reliability.

## Example Configurations

### Production Setup (High Reliability)

```yaml
providers:
  gcp:
    status: prefer
    network_tier: Standard
    ip_version: IPV4
    node_pools:
      baseline: standard
      burst: spot
    public_gateway:
      enabled: true
      static_ip: auto
      http:
        enabled: true
        port: 443

  dfc:
    status: allow

  akash:
    status: fallback
```

### Cost-Optimized Setup

```yaml
providers:
  dfc:
    status: prefer        # Primary for cost
    public_gateway:
      enabled: true

  gcp:
    status: allow         # Secondary for reliability
    network_tier: Standard
    node_pools:
      burst: spot

  akash:
    status: fallback      # Emergency only
```

### Compliance-First Setup (EU Only)

```yaml
providers:
  gcp:
    status: prefer
    public_gateway:
      enabled: true

  dfc:
    status: fallback      # Only EU-based providers

regions:
  geographic_scope: EU    # Restrict to Europe
  count: 3

placement:
  residency:
    require_countries: ["PT", "ES", "FR", "DE", "NL"]
```
