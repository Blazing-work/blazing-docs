Google Cloud Platform (GCP) is a primary provider offering enterprise-grade reliability, global infrastructure, and cost-effective networking options.

## Basic Configuration

```yaml
providers:
  gcp:
    status: prefer
    network_tier: Standard
    ip_version: IPV4
    cert_issuer: letsencrypt-prod
```

## Configuration Reference

### Status Field

```yaml
gcp:
  status: prefer | allow | fallback
```

Define GCP's priority in provider selection:

- **prefer**: Primary provider (use first)
- **allow**: Acceptable provider (use if preferred unavailable)
- **fallback**: Emergency provider (use only when others fail)

### Network Tier

```yaml
gcp:
  network_tier: Standard | Premium
```

GCP offers two network tiers with significant cost differences:

| Tier | Egress Cost | IPv6 | Routing | Best For |
|------|-------------|------|---------|----------|
| **Standard** | **$0.005/GB** | ❌ No | Regional | Cost-optimized (default) |
| **Premium** | $0.085/GB | ✅ Yes | Global | Low-latency worldwide |

**Cost Impact**: Standard tier is **17x cheaper** for egress.

**Example Calculation** (1TB monthly egress):
- Standard: $5/month
- Premium: $85/month
- **Savings: $80/month** (1600% cost reduction)

**When to use Premium**:
- Global user base requiring low latency
- IPv6 requirement
- Real-time applications (gaming, video streaming)

**When to use Standard** (recommended):
- Regional deployments
- Cost-sensitive applications
- Most web/API services

### IP Version

```yaml
gcp:
  ip_version: IPV4 | IPV6
```

**Important**: Standard tier only supports IPv4.

| Combination | Valid | Notes |
|-------------|-------|-------|
| Standard + IPv4 | ✅ | Recommended default |
| Standard + IPv6 | ❌ | Not supported |
| Premium + IPv4 | ✅ | Works but expensive |
| Premium + IPv6 | ✅ | Required for IPv6 |

### Certificate Issuer

```yaml
gcp:
  cert_issuer: letsencrypt-prod | letsencrypt-staging | custom
```

Specify the certificate issuer for TLS:

- **letsencrypt-prod**: Production Let's Encrypt (default)
- **letsencrypt-staging**: Testing (higher rate limits)
- **custom**: Your own certificate authority

## Node Pool Configuration

GKE Standard mode with spot instance support for cost optimization.

```yaml
gcp:
  node_pools:
    baseline: standard      # Always on-demand (read-only)
    burst: spot | standard  # Burst replicas (configurable)
```

### Baseline vs Burst Replicas

**Baseline replicas** (`min_replicas`):
- Always run on **standard (on-demand)** nodes
- Guaranteed availability
- No interruption risk
- Higher cost but reliable

**Burst replicas** (above `min_replicas`):
- Can run on **spot** nodes (if configured)
- 60-90% cost savings
- May be preempted (24h max lifetime)
- Ideal for stateless workloads

### Cost Example

Service with `min_replicas: 3, max_replicas: 10`:

```yaml
services:
  api:
    autoscaling:
      min_replicas: 3    # Baseline (always standard)
      max_replicas: 10   # Up to 7 burst replicas
```

With `node_pools.burst: spot`:

| Scenario | Standard Nodes | Spot Nodes | Cost |
|----------|----------------|------------|------|
| **Baseline (3 replicas)** | 3 | 0 | $0.10/hour |
| **Peak (10 replicas)** | 3 | 7 | $0.17/hour |
| **All standard** | 10 | 0 | $0.33/hour |

**Monthly savings**: ~$350/month (48% reduction)

### Node Pool Configuration

```yaml
gcp:
  node_pools:
    baseline: standard       # Read-only, always standard
    burst: spot             # 60-90% savings on burst
```

Available burst options:

- **spot**: Use spot instances for burst (recommended)
- **standard**: Use on-demand for burst (higher cost, no preemption)

## Public Gateway Configuration

The public gateway provides external connectivity to your services.

```yaml
gcp:
  public_gateway:
    enabled: true
    static_ip: auto | <ip-address>

    # Protocol configuration
    sql:
      enabled: true
      port: 26257

    http:
      enabled: true
      port: 443
      path_prefix: /admin

    # Gateway infrastructure
    gateway:
      replicas: 1
      resources:
        cpu: 500m
        memory: 1Gi
```

### Gateway Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | boolean | false | Deploy public gateway |
| `static_ip` | string | auto | IP allocation (auto or specific) |
| `sql.enabled` | boolean | false | PostgreSQL wire protocol support |
| `sql.port` | number | 5432 | SQL port (e.g., 26257 for CockroachDB) |
| `http.enabled` | boolean | false | HTTP/HTTPS ingress |
| `http.port` | number | 8080 | HTTP port |
| `http.path_prefix` | string | / | Optional path prefix |
| `gateway.replicas` | number | 1 | Gateway replica count |
| `gateway.resources.cpu` | string | 500m | CPU allocation |
| `gateway.resources.memory` | string | 1Gi | Memory allocation |

### SQL Gateway Example

Expose CockroachDB with both SQL and HTTP admin:

```yaml
gcp:
  public_gateway:
    enabled: true
    static_ip: auto

    sql:
      enabled: true
      port: 26257          # CockroachDB SQL port

    http:
      enabled: true
      port: 8080
      path_prefix: /admin  # Admin UI at /admin

    gateway:
      replicas: 1
      resources:
        cpu: 500m
        memory: 1Gi
```

Access:
- SQL: `postgresql://gateway-ip:26257/db`
- Admin UI: `https://gateway-ip/admin`

### HTTP Only Example

Standard web application:

```yaml
gcp:
  public_gateway:
    enabled: true
    static_ip: auto

    http:
      enabled: true
      port: 443

    gateway:
      replicas: 1
      resources:
        cpu: 500m
        memory: 1Gi
```

**Pricing**: $49/month (includes static IP, certificates, load balancing)

## Best Practices

### 1. Use Standard Tier by Default

```yaml
network_tier: Standard  # 17x cheaper egress
```

Only upgrade to Premium if you need:
- IPv6 support
- Global routing with minimal latency
- Real-time gaming or video streaming

### 2. Enable Spot for Burst Workloads

```yaml
node_pools:
  baseline: standard    # Reliable baseline
  burst: spot           # Cost-effective burst
```

This provides:
- Reliability for baseline traffic
- Cost savings for peak traffic
- Automatic fallback to on-demand if spot unavailable

### 3. Single Gateway per Deployment

Enable gateway on only one provider:

```yaml
gcp:
  public_gateway:
    enabled: true       # Main gateway

dfc:
  public_gateway:
    enabled: false      # Workers only
```

Avoid multiple gateways to minimize costs.

### 4. Right-Size Gateway Resources

For typical workloads:

```yaml
gateway:
  replicas: 1
  resources:
    cpu: 500m
    memory: 1Gi
```

Scale up for high-traffic scenarios:

```yaml
gateway:
  replicas: 3
  resources:
    cpu: 1000m
    memory: 2Gi
```

## Troubleshooting

### "IPv6 not supported with Standard tier"

**Error**:
```
ERROR: IPv6 LoadBalancer requires Premium tier
```

**Solution**:
```yaml
gcp:
  network_tier: Standard
  ip_version: IPV4        # Must use IPv4 with Standard
```

### "Node pool exhausted"

**Error**:
```
ERROR: Cannot schedule pod, no nodes available
```

**Solution**: Increase cluster size or reduce replicas:

```yaml
services:
  api:
    autoscaling:
      max_replicas: 20   # Reduce if hitting limits
```

### "Gateway timeout"

**Error**:
```
ERROR: Gateway health check failed
```

**Solution**: Increase gateway resources:

```yaml
gateway:
  replicas: 2            # Add redundancy
  resources:
    cpu: 1000m           # Increase CPU
    memory: 2Gi
```
