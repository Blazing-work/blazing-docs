Core deployments follow a standard lifecycle:

1. **Plan** – review configuration changes
2. **Apply** – deploy via pipeline
3. **Observe** – monitor health and performance
4. **Rollback** – revert when necessary

This workflow is typically automated and owned by the platform team.
