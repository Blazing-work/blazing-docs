**Blazing Core** is the platform layer that manages infrastructure, scaling, and placement for Blazing services.

If you are an application developer, you typically do not interact with Core directly. You write Flow code, publish it, and let your platform team manage Core configuration.

Core is configured via **YAML** in your infrastructure repository and applied through your organization’s deployment pipeline.

## When You Need Core

- You operate the platform infrastructure
- You manage multi‑cloud placement and networking
- You need organization‑wide policies for scaling and access

If you’re unsure, start with [Blazing Flow](/docs/flow-overview).
